# SUAS Website Judge Skill

## Purpose

Evaluate all website decisions, pages, content, navigation, documentation, and user experience according to the official SUAS Team Website rubric.

The objective is to maximize the final website score by ensuring all requirements are satisfied and all categories achieve the **Outstanding** level whenever possible.

---

# 2.4 Team Website

Teams are required to submit a website in English that documents their team, vehicle design, and competition approach.

---

# 2.4.1 Deliverable Requirements

## Website Content

The layout and detailed contents of the website are left to the team; however, the website must include the following:

### Required Team Information

- Current team name
- Contact information

### Required Vehicle Media

- Vehicle photos and/or videos

### Supporting Media

The website should include supporting media such as:

#### Instructional / Informative Videos

- Technical demonstrations
- Educational content
- Process explanations

#### Procedures

May include:

- Text documentation
- Images
- Step-by-step processes

#### Design Decision Documentation

May include:

- Text
- Images
- Videos
- Engineering rationale
- Design tradeoff discussions

#### Historical Records

May include:

- Development blogs
- Build logs
- Progress updates
- Testing history

#### Sponsors

- Sponsor list
- Sponsor logos

---

## Website Quality Requirements

The website should create a positive first impression for:

- Judges
- Sponsors
- Advisors
- Supervisors
- Supporters

The website must be visually appealing and easy to navigate.

### User Experience Requirements

#### Language

- Website must be written in English
- OR provide English translations

#### Content Prioritization

- Important content should be easy to discover
- Key information should be visually prioritized

#### Search

- Site search functionality must be available

#### Design Principles

Use basic design principles including:

- Contrast
- Repetition
- Alignment
- Grouping

These principles should help organize and highlight important content.

#### Accessibility

Must consider accessibility requirements defined by the W3C Web Accessibility Initiative (WAI).

Goals include:

- Keyboard accessibility
- Screen reader compatibility
- Readable content structure
- Accessible color contrast

#### Browser Compatibility

Must function correctly in:

- Chrome
- Firefox
- Safari
- Microsoft Edge

#### Mobile Responsiveness

Must provide a mobile-friendly experience.

# 2.4.2 Scoring Metrics

Total Score: **100 Points**

---

# Team Information (20%)

## Outstanding

The website includes:

- Team name
- Contact information
- Team member list
- Sponsor list

Additionally:

- All vehicle references are relevant to the current competition year

## Strong

The website provides:

- Team introduction
- Team members
- Sponsors

Additionally:

- Vehicle supporting media is available

## Average

The website introduces:

- Team
- And/or team members

## Below Average

The website provides:

- Little or no team information

Additionally:

- No meaningful vehicle information

## Requirements Not Met

Required team information is missing.

---

# Vehicle Design Documentation (40%)

## Outstanding

Vehicle development and testing are thoroughly documented.

Documentation may include:

### Visual Documentation

- Photographs
- Diagrams
- Videos

### Technical Documentation

- Procedures (text + images)
- Design documentation (text + images + video)
- Engineering explanations

### Historical Documentation

- Build logs
- Development blogs
- Testing records

Documentation should clearly explain:

- Development process
- Testing process
- Engineering decisions
- Lessons learned

## Strong

The website provides:

- Good vehicle development documentation
- Good testing documentation

Additionally:

- Supporting media is accessible

## Average

The website adequately presents:

- Vehicle development process
- Vehicle testing process

With some supporting media.

## Below Average

The website contains:

- A few photos or videos

But lacks:

- Instructional documentation
- Informative documentation

## Requirements Not Met

No vehicle visuals or documentation are available.

---

# Website Quality (40%)

## Outstanding

The website demonstrates strong human-centered design.

Characteristics include:

- Visually appealing layout
- Clear visual hierarchy
- Excellent usability
- Intuitive navigation
- Fast access to important information
- Strong content organization

The website consistently guides user attention toward relevant content.

## Strong

The website demonstrates good user experience.

Characteristics include:

- Easy navigation
- Good visual hierarchy
- Efficient information discovery

## Average

The website is usable.

Users can locate most information without significant difficulty.

## Below Average

Problems may include:

- Poor layout
- Confusing organization
- Difficult navigation
- Weak visual hierarchy

The website is not user-friendly.

## Requirements Not Met

The website is:

- Cluttered
- Difficult to read
- Difficult to navigate

No clear guidance is provided for finding information.

---

# Evaluation Rules

When reviewing any page, feature, or design decision:

1. Identify which rubric category is affected.
2. Estimate the impact on the score.
3. Identify missing Outstanding-level requirements.
4. Recommend improvements.
5. Never assume a requirement is satisfied without evidence.
6. Prioritize rubric compliance over aesthetics.
7. Prioritize usability over visual effects.
8. Prioritize documentation completeness over decorative content.

---

# Success Criteria

The website should target:

- Outstanding Team Information
- Outstanding Vehicle Design Documentation
- Outstanding Website Quality

Target Score:

**95–100 / 100**
